# User Experience Index (UXI)

# 詳細解説

---

# 1. UXIとは何か？

UXI（User Experience Index）は、

> 「複数のUX指標を統合し、UX価値を単一スコアで表現する」

ための評価手法である。

従来のUX評価では、

* Attractiveness（魅力）
* Efficiency（効率）
* Perspicuity（分かりやすさ）
* Dependability（信頼性）
* Stimulation（刺激・楽しさ）
* Novelty（新規性）

など複数の尺度を個別に評価していた。

しかし実務では、

```text
Attractiveness = 1.9
Efficiency = 2.1
Novelty = 0.8
```

のような結果を見ても、

> 「結局この製品のUXは良いのか？」

が直感的に分かりにくい。

UXIはこれを解決するために提案された。

---

# 2. 背景

UX研究では長年、

## Usability

* 操作性
* 効率性
* エラー率

を中心に評価していた。

しかし近年では、

## User Experience

* 感情
* 楽しさ
* 快適性
* 信頼感
* 没入感

も重要になっている。

その結果、

UX評価項目が増えすぎてしまった。

---

# 3. UXIの目的

UXIの目的は、

```text
複数のUX指標
      ↓
統合
      ↓
単一スコア
```

である。

例えば：

```text
UXI = 82.4
```

のように表現できる。

---

# 4. 元になった評価手法

UXIは主に

## User Experience Questionnaire（UEQ）

を基盤としている。

---

# 5. UEQの6尺度

| 尺度             | 内容     |
| -------------- | ------ |
| Attractiveness | 魅力     |
| Perspicuity    | 分かりやすさ |
| Efficiency     | 効率     |
| Dependability  | 信頼性    |
| Stimulation    | 楽しさ    |
| Novelty        | 新規性    |

---

# 6. なぜ単純平均ではダメなのか？

例えば：

| 項目             | 点数 |
| -------------- | -: |
| Attractiveness | 90 |
| Efficiency     | 95 |
| Novelty        | 20 |

平均：

```text
(90+95+20)/3
=68.3
```

しかし実際には、

用途によって重要度が異なる。

---

## 自動車HMI

重要：

* Efficiency
* Dependability

---

## ゲーム

重要：

* Stimulation
* Novelty

---

## ECサイト

重要：

* Perspicuity
* Efficiency

---

つまり、

UX要素は同じ価値ではない。

---

# 7. UXIの特徴

UXIでは、

各項目へ重みを与える。

---

## イメージ

```text
UXI

=
0.25 × Efficiency
+
0.20 × Dependability
+
0.18 × Attractiveness
+
0.15 × Perspicuity
+
0.12 × Stimulation
+
0.10 × Novelty
```

---

# 8. Fuzzy Measureとは？

論文最大の特徴。

---

## なぜ必要？

UXは曖昧だから。

例えば：

```text
とても良い
まあ良い
少し良い
```

は厳密な数値ではない。

---

## Fuzzy Logic

これを

```text
0.9
0.7
0.5
```

のような連続値として扱う。

---

# 9. Gaussian Fuzzy Number

論文では、

回答値を

## Gaussian Fuzzy Number

へ変換する。

---

イメージ：

```text
「良い」
```

↓

```text
中心値 0.8
ばらつき 0.1
```

↓

曖昧さ込みで扱う

---

# 10. UXI算出フロー

```text
UEQ回答
    ↓
Fuzzy変換
    ↓
重み付け
    ↓
統合
    ↓
UXI算出
```

# 11. 実験対象

---

* 実際に使用

製品比較

---

# 12. UXIのメリット

## ① 製品比較しやすい

例：

| 製品 | UXI |
| -- | --: |
| A  |  82 |
| B  |  74 |
| C  |  68 |

---

## ② KPI化できる

例：

```text
UXI > 80
```

を開発目標に設定可能。

---

## ③ 改善効果が分かる

```text
改善前 68
改善後 81
```

のように比較できる。

---

# 13. UXIの限界

## 情報圧縮

単一スコア化すると、

詳細情報が失われる。

---

例：

```text
UXI = 80
```

でも、

* 効率が高い
* 楽しさが低い

かもしれない。

---

## 重み依存

重み設定によって結果が変わる。

---

## 業界依存

重要なUX要素は業界ごとに異なる。

---

# 14. 自動車UXへの応用

UXIの考え方は、

自動車UXへ非常に応用しやすい。

---

## Automotive UX Index（例）

```text
AUXI

=
0.20 × 満足度
+
0.20 × 操作成功率
+
0.15 × 操作時間
+
0.15 × HRV
+
0.10 × 呼吸
+
0.10 × 瞬目
+
0.05 × 姿勢変化
+
0.05 × 脳血流
```

# 18. まとめ

UXI（User Experience Index）は、

> 「複数のUX指標を1つのスコアへ統合する」

ための代表的研究である。

特徴は：

* UEQベース
* Fuzzy Logic活用
* 重み付け統合
* KPI化可能

である。

また、

自動車UXや生理信号を活用した評価へ拡張しやすく、

将来的には

* HRV
* 呼吸
* 脳血流
* 瞬目
* AI感情推定

などを統合した

次世代UXIへ発展する可能性が高い。
